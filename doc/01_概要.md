# MetricFlow 評価環境 - プロジェクト概要

## このプロジェクトは何か

このプロジェクトは **dbt** と **MetricFlow** を使って、ECサイトの売上・顧客データを集計・分析するための学習・評価環境です。  
サンプルデータ（顧客・注文・支払い）を使って、データの変換から指標（メトリクス）定義までの一連の流れを体験できます。

---

## 使われている技術

| 技術 | 役割 | バージョン |
|------|------|-----------|
| **PostgreSQL** | データを保存するデータベース | 16 |
| **dbt** | SQLを使ってデータを変換・加工するツール | 1.8.x |

| **Docker** | 全サービスをまとめて起動するコンテナ環境 | - |
| **pgAdmin** | ブラウザでDBを閲覧できる管理ツール | 4 |

---

## dbt とは

**dbt（data build tool）** はデータを変換するためのツールです。  
「生のデータ」を「分析しやすい形」に加工するSQL処理を、バージョン管理・テスト・ドキュメントつきで管理できます。

```
生データ（CSV）  →  dbt  →  きれいな分析用テーブル（PostgreSQL）
```

dbt の特徴：
- **SQLだけで書ける** — Pythonや複雑なスクリプトは不要
- **依存関係を自動解決** — 「このテーブルを作る前にあのテーブルを作る」を自動判断
- **テスト内蔵** — データ品質チェック（NULL禁止、重複禁止など）を宣言的に定義
- **再現性** — 何度実行しても同じ結果になる

---

## MetricFlow とは

**MetricFlow** は「売上合計」「注文件数」などのビジネス指標を **一箇所に定義** して、SQLを自動生成してくれるフレームワークです。

通常、指標を計算するSQLはレポートやダッシュボードごとにバラバラに書かれがちです。  
MetricFlowでは指標をYAMLファイルで定義することで：

- 定義が一元化される（「売上」の計算方法が一種類になる）
- 「月別売上」「国別売上」など **集計軸（ディメンション）を自由に組み合わせられる**
- SQLを手書きしなくてよい

## プロジェクト全体のデータフロー

```
seeds/（CSVファイル）
    │
    ▼  dbt seed
┌─────────────────────────────────────────┐
│  rawスキーマ（PostgreSQL）              │
│  ├── raw_customers  （顧客原始データ）  │
│  ├── raw_orders     （注文原始データ）  │
│  └── raw_payments   （支払原始データ）  │
└─────────────────────────────────────────┘
    │
    ▼  dbt run（staging層）
┌─────────────────────────────────────────┐
│  analyticsスキーマ - ステージング（ビュー）│
│  ├── stg_customers  （顧客クレンジング）│
│  ├── stg_orders     （注文クレンジング）│
│  └── stg_payments   （支払クレンジング）│
└─────────────────────────────────────────┘
    │
    ▼  dbt run（marts層）
┌─────────────────────────────────────────┐
│  analyticsスキーマ - マート（テーブル）  │
│  ├── customers          （顧客集計表）  │
│  ├── orders             （注文集計表）  │
│  └── metricflow_time_spine（日付一覧）  │
└─────────────────────────────────────────┘
    │
    ▼  MetricFlow定義（YAMLで宣言）
┌─────────────────────────────────────────┐
│  メトリクス（12種類）                   │
│  ├── revenue（売上合計）                │
│  ├── order_count（注文件数）            │
│  ├── completion_rate（注文完了率）      │
│  └── ... その他9種類                   │
└─────────────────────────────────────────┘
```

## ファイル構成の地図

```
metricflow/
├── docker-compose.yml          ← Docker起動設定
├── docker/
│   ├── Dockerfile.dbt          ← dbt実行環境のビルド定義
│   └── postgres-init/
│       └── 01_setup.sql        ← DB起動時にスキーマを作成
├── dbt_project/                ← dbtプロジェクトの本体
│   ├── dbt_project.yml         ← プロジェクト全体の設定
│   ├── profiles.yml            ← DB接続情報
│   ├── packages.yml            ← 外部ライブラリ定義
│   ├── seeds/                  ← 投入するCSVデータ
│   ├── models/                 ← SQLモデル定義
│   │   ├── staging/            ← 第1層：生データ整形
│   │   ├── marts/              ← 第2層：ビジネスロジック
│   │   └── metrics/            ← MetricFlow定義
│   └── tests/                  ← カスタムテスト（今は空）
└── scripts/
    ├── setup.ps1 / setup.sh    ← 初期セットアップスクリプト
    └── run_queries.ps1 / .sh   ← メトリクスクエリ実行スクリプト
```
